# Playbook & Reference Guidelines

> [!NOTE]
> These guidelines serve as an aspirational, reference "platform playbook" for B2B/SaaS architectures. 
> They are templates to draw conventions and best practices from, rather than strict constraints for this specific repository. 
> When implementing, always align with the project's actual stack (e.g., Next.js App Router, Drizzle ORM, single-tenant) rather than forcing Express/Prisma/multi-tenant patterns.
